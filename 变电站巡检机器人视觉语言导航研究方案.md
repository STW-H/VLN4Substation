# 面向变电站巡检的安全约束主动视觉语言观测导航研究方案

## 一、研究背景

变电站巡检机器人是电力系统智能运维的重要装备，可承担设备外观检查、表计读取、红外测温、接头过热检测和异常状态识别等任务。现有巡检机器人通常依赖离线地图、激光雷达、视觉定位、RTK、SLAM、全局路径规划和 DWA/TEB 等局部规划算法完成导航，其优势在于几何路径规划、避障、路径跟踪和安全距离控制。

然而，变电站巡检任务的目标并不只是“到达某个固定坐标点”，而是“到达能够完成巡检作业的观测位姿”。例如，用户可能只输入：

```text
检查 1 号主变 A 相接头是否过热。
```

传统导航系统可以根据离线地图将机器人送至预设巡检点，但固定巡检点在实际场景中可能存在以下问题：

- 接头、仪表或铭牌被设备结构、围栏、支撑架、临时工具遮挡；
- 机器人到达预设点后，目标部件不在相机或红外视野中心；
- 拍摄角度不适合红外测温或图像识别；
- 目标距离过远、过近或视角偏斜，导致图像质量不足；
- 现场状态与离线建图时不同，固定机位难以保证巡检结果可靠。

因此，本研究不再将 VLN 的主要作用定义为全局路径规划或常规避障。路径规划、路径跟踪和安全距离控制仍由离线地图、在线局部规划器和安全约束地图完成；视觉语言导航或视觉语言模型的核心作用是：在候选巡视区域内，根据自然语言巡检任务和当前 RGB/红外/深度观测，主动调整最终观测位姿，使机器人从“到达候选点”进一步达到“看得清、看得准、可检测”的巡检作业状态。

本研究拟提出一种面向变电站巡检的安全约束主动视觉语言观测导航方法。该方法利用 3D Gaussian Splatting 构建高保真视觉仿真场景，利用点云、NavMesh、语义地图和安全约束地图生成安全可达的候选巡视点或候选观测区域，再引入现有 VLN/VLM 作为黑盒主动观测模块，在候选区域内完成局部视角调整、目标可见性判断和巡检图像采集位姿优化。

该方案的核心定位是：

```text
传统导航解决“安全到附近”；
VLN/VLM 解决“是否看清、如何调整到可巡检视角”；
安全地图解决“哪些区域不能进入、哪些距离不能突破”。
```

## 二、研究目标

本研究的总体目标是构建一种面向变电站巡检任务的安全约束主动视觉语言观测导航框架，使机器人能够根据用户自然语言巡检意图，在安全地图和传统导航系统的支持下到达候选巡视区域，并利用 VLN/VLM 根据现场视觉反馈自适应优化最终观测位姿，从而获得满足巡检检测要求的 RGB/红外图像。

具体目标包括：

1. 构建 3DGS 与点云地图统一的空间表达  
   利用变电站 3D Gaussian Splatting 场景和点云地图，通过尺度校正与配准实现视觉渲染场景、几何导航地图、设备语义和安全约束的统一坐标表达。

2. 构建变电站语义地图与安全约束地图  
   建立包含设备、部件、巡检通道、禁入区、安全缓冲区、候选观测区域和巡检目标的结构化地图，为候选巡视点生成和安全检查提供基础。

3. 实现自然语言巡检任务到设备/部件目标的解析  
   将“检查某接头是否过热”“查看某仪表读数”等任务解析为目标设备、目标部件、传感器需求、候选观测区域和检测质量要求。

4. 设计安全约束下的候选巡视点或候选观测区域生成方法  
   根据设备位置、部件朝向、安全距离、可通行区域、巡检通道和传感器视场，生成若干安全可达的候选观测点或局部观测区域。

5. 利用传统导航与在线局部规划实现候选区域到达  
   通过 A*/Dijkstra/拓扑规划生成全局路线，并通过 DWA/TEB 或 Habitat-GS 中的局部控制策略完成路径跟踪、避障和安全距离控制。

6. 引入 VLN/VLM 实现局部主动观测位姿优化  
   在候选巡视区域内，VLN/VLM 根据巡检任务语言、当前 RGB/红外/深度观测和候选区域约束，判断目标是否可见、是否无遮挡、是否满足检测要求，并输出小范围移动、转向、抬头/低头、换候选点或停止采集等动作。

7. 建立任务完成质量评价体系  
   不仅评价导航是否到达，还评价目标可见性、遮挡情况、成像质量、红外测温可用性、安全距离满足情况和最终巡检图像质量。

## 三、研究内容与技术路线

本研究的核心思路是：将 3DGS 用作高保真视觉环境，将点云/NavMesh 用作几何导航基础，将语义地图和安全规则用于候选观测区域生成，将传统导航用于将机器人送至候选巡视区域，将 VLN/VLM 用于候选区域内的主动观测位姿优化。

整体流程如下：

```text
自然语言巡检任务
→ 任务解析与目标部件 grounding
→ 语义地图与安全约束地图查询
→ 候选巡视点/候选观测区域生成
→ 离线全局规划与在线局部规划到达候选区域
→ 3DGS 第一人称视觉/红外/深度观测
→ VLN/VLM 判断目标可见性与图像质量
→ 安全约束下局部观测位姿调整
→ 获取满足巡检要求的图像
→ 巡检检测与结果评价
```

### 3.1 3DGS 与点云地图空间对齐

3DGS 场景主要用于第一人称视觉渲染，点云或 NavMesh 主要用于几何导航、安全约束和候选区域生成，因此需要将二者统一到同一坐标系下。

具体步骤包括：

- 从 3DGS 文件中提取 Gaussian 中心点，形成 GS-derived point cloud；
- 对高斯中心点进行过滤，去除异常尺度、低透明度或离群点；
- 将高斯中心点云与变电站点云进行人工粗配准；
- 选取道路拐角、设备基础角点、围栏角点、建筑物角点等同名特征点；
- 使用 ICP 或相似变换估计尺度、旋转和平移；
- 将设备位置、禁入区、安全缓冲区、候选观测点统一转换到 Habitat-GS 坐标系。

若 3DGS 来源于单目图像重建，其尺度可能不等于真实尺度，需要利用道路宽度、设备基础尺寸或已知测距数据进行尺度校正。

### 3.2 可通行地图与传统导航层

传统导航层负责“将机器人安全送至候选巡视区域”。该部分不由 VLN 承担，而由成熟地图导航方法完成。

可通行区域包括：

- 主巡视通道；
- 设备间通道；
- 站内道路；
- 主控楼前区域；
- 巡检机器人允许进入的安全作业区域。

不可通行区域包括：

- 设备基础；
- 围栏内部；
- 沟槽、台阶、墙体等障碍区域；
- 带电设备禁入区；
- 临时检修封闭区域。

可采用二维栅格地图、拓扑图或 Habitat-GS NavMesh 表达可通行空间。全局规划可使用 A*、Dijkstra 或拓扑搜索，局部规划可采用 DWA、TEB、MPC 或 Habitat 仿真动作控制。该层主要评价路径长度、避障能力、最小安全距离和到达候选区域的成功率。

### 3.3 语义地图与巡检目标建模

为了将用户任务映射到设备空间，需要构建变电站语义地图。语义地图不仅包含设备整体位置，还应包含可巡检部件、观测需求和候选观测区域。

语义元素包括：

- 设备实体：主变、断路器、隔离开关、避雷器、互感器、开关柜、GIS 设备等；
- 设备部件：主变高压侧、低压侧、中性点侧、A/B/C 相接头、套管、油位表、温度表、避雷器计数器、断路器操作机构等；
- 巡检任务类型：红外测温、表计读取、外观检查、状态确认、铭牌识别；
- 候选观测区域：围绕目标部件、满足安全距离和可达性的可观察区域；
- 观测质量条件：目标可见、无遮挡、位于视野中心附近、大小合适、红外区域完整可测温。

示例结构如下：

```json
{
  "objects": [
    {
      "id": "T1_A_joint",
      "name": "1号主变A相接头",
      "equipment": "1号主变",
      "part_type": "joint",
      "task_types": ["thermal_inspection"],
      "position": [12.5, 2.1, -8.3],
      "safe_distance": 1.5,
      "candidate_observation_region": "T1_A_joint_obs_region",
      "visual_cues": ["主变高压侧", "A相套管附近", "金属接头"],
      "quality_requirements": {
        "target_visible": true,
        "thermal_region_complete": true,
        "min_bbox_ratio": 0.04,
        "max_occlusion_ratio": 0.3
      }
    }
  ]
}
```

### 3.4 安全约束地图构建

安全约束地图用于限制候选观测区域和局部位姿调整范围。主要约束包括：

- 禁入区约束：机器人不得进入围栏内部、带电间隔、检修封闭区域；
- 安全距离约束：机器人与带电设备、高压区域或危险设备保持最小安全距离；
- 通道约束：机器人应优先在主巡视通道、设备间通道或允许作业区域内移动；
- 视线约束：候选观测区域应尽量保证目标部件处于传感器可观察范围；
- 运动边界约束：VLN/VLM 的局部调整动作只能在安全候选区域内执行。

安全约束地图不只参与路径规划，还参与 VLN/VLM 输出动作的过滤。若模型建议前进、侧移或换点观察，但该动作会突破禁入区、安全距离或可通行边界，则系统拒绝该动作，并由安全规划器给出替代动作。

### 3.5 自然语言巡检任务解析

本研究不要求用户提供坐标或路径，而是输入巡检意图，例如：

```text
检查 1 号主变 A 相接头是否过热。
查看 2 号避雷器计数器读数。
确认 3 号断路器分合闸状态。
```

系统将任务解析为结构化表示：

```json
{
  "task": "thermal_inspection",
  "target_equipment": "1号主变",
  "target_part": "A相接头",
  "sensor": ["rgb", "thermal"],
  "quality_goal": "目标接头在红外图像中完整可见，适合测温"
}
```

解析方式可采用模板规则、关键词匹配、设备台账匹配或通用 LLM API。解析结果与语义地图进行 grounding，确定目标部件位置、候选观测区域和安全约束。

### 3.6 候选巡视点与候选观测区域生成

候选巡视点不再只是固定终点，而是一个满足安全和观测条件的候选集合。生成时综合考虑：

- 目标部件位置与朝向；
- 相机/红外传感器视场角；
- 目标距离范围；
- 设备安全距离；
- 可通行区域；
- 巡检通道偏好；
- 视线遮挡情况；
- 机器人运动约束。

可采用两种表示方式：

1. 离散候选点  
   在目标周围安全区域采样若干候选点，每个点包含位置、朝向、预估可见性和安全代价。

2. 连续候选区域  
   生成一个局部可移动区域，传统导航先到达该区域，VLN/VLM 在区域内主动调整最终观测位姿。

候选点可打分：

```text
S(v) = w1 * Visibility(v) + w2 * ImageQuality(v) - w3 * Risk(v) - w4 * PathCost(v)
```

其中 Visibility 表示目标可见性，ImageQuality 表示成像质量估计，Risk 表示安全风险，PathCost 表示到达代价。

### 3.7 传统导航到达候选区域

在候选区域确定后，系统使用传统导航方法将机器人送至候选区域附近。该阶段可输出：

- 全局安全路径；
- 局部避障轨迹；
- 候选区域入口点；
- 到达候选区域的终止条件。

该阶段不强调 VLN 的参与，因为 DWA/TEB 等局部规划算法已经能够较好处理路径跟踪、动态避障和安全距离控制。这样可以避免 VLN 与传统导航功能重复，使 VLN 的职责更集中。

### 3.8 VLN/VLM 主动观测位姿优化

机器人到达候选区域后，启动 VLN/VLM 主动观测模块。该模块接收：

- 当前 RGB 图像；
- 可选红外图像；
- 可选深度图；
- 当前机器人位姿；
- 当前候选区域边界；
- 巡检任务语言；
- 目标部件视觉描述；
- 安全约束提示。

输出动作可以是离散动作：

```text
turn_left
turn_right
move_forward
move_backward
move_left
move_right
look_up
look_down
switch_candidate_viewpoint
capture
stop
```

也可以输出高层建议：

```text
目标在画面右侧且被遮挡，建议向右侧候选点移动后重新观察。
```

VLN/VLM 的主要判断包括：

- 目标部件是否出现在视野中；
- 目标是否被遮挡；
- 目标是否位于合适的图像区域；
- 目标在红外图像中是否完整可测温；
- 当前视角是否满足巡检图像采集要求；
- 是否需要调整位置、朝向或切换候选点。

该模块的本质不是替代 DWA，而是将终点从“固定坐标位姿”优化为“满足巡检任务的观测位姿”。

### 3.9 安全过滤与动作接管

VLN/VLM 输出的局部动作不能直接执行，需要经过安全过滤。过滤条件包括：

- 是否离开候选安全区域；
- 是否进入禁入区；
- 是否低于带电设备安全距离；
- 是否与障碍物或设备基础碰撞；
- 是否超出机器人可执行动作范围；
- 是否导致观测任务无效或远离目标。

若动作不安全，系统采取以下策略：

- 拒绝该动作；
- 使用局部规划器生成替代动作；
- 切换到另一个候选观测点；
- 触发重新规划候选区域；
- 请求人工确认。

### 3.10 3DGS/Habitat-GS 仿真验证

在 Habitat-GS 中导入 3DGS 场景和 NavMesh，使机器人能够在高保真视觉场景中执行候选区域到达和局部观测调整。

仿真过程包括：

- 加载 3DGS 场景；
- 加载 NavMesh、语义地图和安全约束地图；
- 根据任务生成目标部件和候选观测区域；
- 传统导航到达候选区域；
- 渲染第一人称 RGB、深度和可选红外模拟图；
- VLN/VLM 进行局部观测位姿调整；
- 保存最终采集图像、调整轨迹和安全约束检查结果；
- 可视化候选点、最终观测点、禁区、安全缓冲区和目标视线。

## 四、研究创新性

### 4.1 从固定巡检点导航转向任务可观测位姿优化

传统巡检机器人通常以固定坐标点和固定朝向作为导航终点。本研究将终点定义为“满足巡检检测要求的观测位姿”，评价机器人是否真正看清目标、是否适合红外测温或视觉识别，而不仅是是否到达坐标点。

### 4.2 安全地图、传统导航与 VLN/VLM 的职责解耦

本研究不让 VLN/VLM 承担全局规划、常规避障和安全距离控制，而是由安全地图和传统导航层完成这些高可靠任务。VLN/VLM 专门负责语义目标确认、可见性判断和观测位姿调整，避免与成熟导航算法重复。

### 4.3 面向变电站巡检的主动视觉语言观测机制

针对接头过热检测、表计读取和开关状态确认等任务，设计基于视觉语言反馈的局部主动观察机制，使机器人能够根据现场图像质量自适应调整位置和视角，解决固定机位在遮挡、角度偏差和现场变化下拍摄不清晰的问题。

### 4.4 安全约束下的候选观测区域生成

结合设备位置、安全距离、禁入区、传感器视场和目标可见性生成候选观测区域，使机器人既能满足变电站安全规程，又能获得有效巡检图像。

### 4.5 不依赖端到端 VLN 训练的轻量接入路线

本研究可先将现有 VLM/VLN 或 API 模型作为黑盒主动观测模块使用，不需要立即训练完整端到端 VLN 模型。若后续需要提升稳定性，可针对观测质量判断和局部调整策略进行小规模微调。

## 五、软硬件环境

### 5.1 硬件环境

当前可采用的硬件环境如下：

- GPU：NVIDIA RTX 4060；
- CPU：AMD Ryzen 9 7945HX；
- 内存：32 GB RAM；
- 存储：建议预留 100 GB 以上空间，用于存放 3DGS 场景、点云、渲染图像和实验结果；
- 操作系统：Ubuntu 24.04。

## 六、实施过程

### 阶段一：数据准备

- 准备变电站 3DGS 场景文件，如 `substation.gs.ply`；
- 准备对应点云文件或 NavMesh；
- 准备设备台账、设备部件位置和巡检任务类型；
- 准备安全区域、禁入区和巡检通道标注；
- 准备少量自然语言巡检任务。

### 阶段二：3DGS 与点云地图配准

- 从 3DGS 中提取高斯中心点；
- 与真实点云或地图进行粗配准；
- 使用 ICP 或相似变换进行精配准；
- 统一设备、候选点、禁区和安全缓冲区坐标。

### 阶段三：构建语义地图与安全约束地图

- 标注设备实体和关键部件；
- 标注接头、仪表、开关状态观察区域等巡检目标；
- 标注禁入区、安全缓冲区和主巡视通道；
- 保存为 `semantic_map.json` 和 `constraint_map.json`。

### 阶段四：构建巡检任务集

设计 20 至 50 条自然语言巡检任务，例如：

- “检查 1 号主变 A 相接头是否过热。”
- “查看 2 号避雷器计数器读数。”
- “确认 3 号断路器是否处于分闸状态。”
- “检查 1 号主变油位表读数是否正常。”

每条任务标注目标设备、目标部件、传感器类型、候选观测区域和图像质量要求。

### 阶段五：候选巡视点或观测区域生成

- 根据目标部件位置生成安全距离约束；
- 在可通行区域内采样候选观测点；
- 根据视场、距离、朝向和遮挡估计候选点质量；
- 过滤不安全或不可达候选点；
- 输出候选点集合或候选观测区域。

### 阶段六：传统导航到达候选区域

- 加载可通行地图或 NavMesh；
- 使用 A*/Dijkstra/拓扑规划生成到候选区域的路径；
- 使用 DWA/TEB 或仿真动作控制到达候选区域；
- 记录路径长度、到达时间、最小安全距离和避障情况。

### 阶段七：VLN/VLM 主动观测位姿调整

- 获取当前 RGB/深度/红外观测；
- 将巡检任务、目标描述和当前图像输入 VLN/VLM；
- 输出局部观测调整动作或换点建议；
- 通过安全过滤模块检查动作；
- 执行动作并循环观察；
- 当目标满足可见性和图像质量要求时执行 `capture/stop`。

### 阶段八：观测质量标注与可选训练数据准备

若现有 VLN/VLM 对变电站任务不稳定，可准备针对性数据：

- 多视角 RGB/红外/深度图像；
- 相机位姿和机器人位姿；
- 目标是否可见；
- 目标是否完整；
- 遮挡比例；
- 目标是否居中；
- 红外测温区域是否完整；
- 图像质量评分；
- 从不理想视角到理想视角的局部调整动作序列。

优先训练或微调两个轻量模块：

1. 观测质量评估器：`image + task → quality score`；
2. 局部视角调整策略：`image + task + pose → action`。

不建议一开始训练完整端到端 VLN。

### 阶段九：Habitat-GS 可视化仿真

- 导入 3DGS 场景和 NavMesh；
- 可视化候选观测点、安全缓冲区和禁入区；
- 渲染机器人第一人称视角；
- 保存局部调整轨迹、关键帧和最终采集图像；
- 生成实验视频和俯视图轨迹。

### 阶段十：实验评价

评价指标包括：

- 候选区域到达成功率；
- 最终观测成功率；
- 目标可见率；
- 目标完整可见率；
- 平均遮挡比例；
- 目标中心偏差；
- 目标图像尺寸占比；
- 红外测温区域有效率；
- 最终图像质量评分；
- 局部调整步数；
- 切换候选点次数；
- 安全约束违规次数；
- VLN/VLM 动作被安全过滤次数；
- 最小安全距离；
- 任务完成时间。

对比方法可设置为：

- 固定预设巡检点；
- 离线地图生成的最佳候选点但不进行主动调整；
- 仅传统导航到候选区域后固定拍摄；
- 本文方法：传统导航到候选区域 + VLN/VLM 主动观测位姿优化；
- 可选：使用规则式视觉质量判断替代 VLM 的消融方法。

预期结果是：本文方法相比固定巡检点能够提高目标可见率、图像质量和红外测温有效率；相比只依赖传统导航的方法，能够在遮挡、视角偏差和现场变化情况下更灵活地获得可用巡检图像，同时保持安全距离和禁区约束。

## 七、可省略与保留模块说明

为降低实现难度，前期可以省略以下模块：

- 不训练完整端到端 VLN 大模型；
- 不修改现有 VLN/VLM 模型结构；
- 不做在线 SLAM；
- 不做真实机器人部署；
- 不做复杂动态障碍物建模；
- 不做机械臂操作；
- 不做大规模开放词汇 3D 语义建图。

但应保留以下关键要素：

- 自然语言巡检任务输入；
- 3DGS 第一人称视觉观测；
- 设备/部件语义地图；
- 安全约束地图；
- 候选巡视点或候选观测区域生成；
- 传统导航到达候选区域；
- VLN/VLM 基于视觉反馈进行局部观测位姿调整；
- 观测质量评价；
- 安全过滤与动作接管；
- Habitat-GS 可视化仿真。

因此，本研究不是传统意义上的端到端 VLN，也不是普通路径规划，而是面向变电站巡检任务的安全约束主动视觉语言观测导航方法。

## 八、预期成果

本研究预期形成以下成果：

- 一套变电站 3DGS 场景与点云/NavMesh 配准流程；
- 一套面向巡检任务的设备部件语义地图与安全约束地图；
- 一种自然语言巡检任务到目标部件和观测需求的解析方法；
- 一种安全约束下的候选巡视点/候选观测区域生成方法；
- 一个传统导航到达候选区域 + VLN/VLM 主动观测位姿优化的仿真系统；
- 一套观测质量评价指标，包括目标可见性、遮挡、图像质量和红外测温可用性；
- 若干可视化实验结果，包括候选点分布图、第一人称视角图、局部调整轨迹、最终采集图像和实验视频；
- 一篇面向电力巡检机器人、机器人主动感知或电力具身智能方向的研究论文。

## 九、总结

本研究面向变电站巡检机器人任务，提出一种安全约束主动视觉语言观测导航方法。该方法不将 VLN/VLM 用作全局路径规划器，也不让其替代 DWA/TEB 等成熟局部规划算法，而是将其用于传统导航最难解决的环节：在候选巡视区域内根据任务语言和现场视觉反馈主动优化最终观测位姿。

该方案利用 3DGS 构建高保真仿真场景，利用点云、NavMesh 和安全地图保证几何可达与安全约束，利用语义地图将用户自然语言巡检任务映射到设备部件和候选观测区域，并利用 VLN/VLM 判断目标可见性、遮挡情况和图像质量，从而实现从“到达固定巡检点”到“到达可完成巡检任务的观测位姿”的转变。

该路线避免了大规模端到端 VLN 训练和真实机器人实验的高成本问题，适合先在 RTX 4060 与 Habitat-GS 环境下完成中小规模仿真验证。其主要贡献不在于训练新的视觉语言模型，而在于面向变电站巡检场景重新定义 VLN/VLM 的合理作用，将安全地图、传统导航和视觉语言主动观测结合起来，形成一种更符合工业巡检需求的轻量级、可实现、可验证的技术方案。
